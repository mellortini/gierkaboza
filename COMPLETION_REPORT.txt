                            CORE FEATURES
================================================================================

GLOBAL CLOCK SYSTEM
  • Single source of truth: world.currentTimeMinutes
  • Integer-based (no floating point time)
  • Arbitrary epoch (starts at 0)
  • Only modified via advanceWorldTime()
  • Validates against negative values

TIME ADVANCEMENT
  • advanceWorldTime(minutes) method
  • Automatic system updates
  • Correct execution order
  • Event queue placeholder (Phase 2)

TIME-DEPENDENT SYSTEMS
  • HP Regeneration: 1 per minute (configurable)
  • Stamina Regeneration: 2 per minute (configurable)
  • Mana Regeneration: 0.5 per minute (configurable)
  • Hunger Consumption: 0.5 per minute (configurable)
  • Thirst Consumption: 1.0 per minute (configurable)
  • Fatigue Consumption: 0.3 per minute (configurable)
  • Status Effect Duration Tracking
  • Automatic Status Effect Application
